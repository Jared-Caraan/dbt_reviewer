What is the primary benefit of Jinja in a SQL workflow?

a. To handle database connections\
b. To add security layers to SQL queries\
c. To make SQL code reusable and dynamic through templating\
d. To optimize query performance through indexing

Answer: c

##
Which of the following is a valid syntax for setting a variable in Jinja?

a. `{% set my_variable = "value" %}`\
b. `{% let my_variable = "value" %}`\
c. `{% create my_variable = "value" %}`\
d. `{{ set my_variable = "value" }}`

Answer: a

##
What will be the output of this Jinja block:

```jinja
{% set my_temperature = 80 %}

{% if my_temperature > 70 %}

    {{ "Iced Tea" }}

{% else %}

    {{ "Hot Tea" }}

{% endif %}
```

a. "Hot Tea"\
b. "Iced Tea"\
c. ERROR\
d. "80"

Answer: b
