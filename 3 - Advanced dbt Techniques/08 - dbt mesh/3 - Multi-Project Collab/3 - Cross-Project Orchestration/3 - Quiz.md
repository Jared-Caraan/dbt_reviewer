Cross-project orchestration allows you to...

a. run jobs on a schedule in more than one project.\
b. trigger a job to run in a project after an upstream project's job has run.\
c. trigger a job to run in a project after a downstream project's job has run.\
d. run jobs on a schedule in your project.

Answer: b

##
How do you set up a job in a downstream project to run after a job in an upstream project has run?

a. This is impossible.\
b. In an upstream project, create a job that runs your downstream project's code.\
c.In an upstream project's job, enable the 'Run when another job finishes' option and specify the downstream job that, when it completes, will trigger your job.\
d.In a downstream project's job, enable the 'Run when another job finishes' option and specify the upstream job that, when it completes, will trigger your job.

Answer: d
