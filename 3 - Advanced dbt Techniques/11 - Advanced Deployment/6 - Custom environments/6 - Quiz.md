What is the primary purpose of using environment variables in dbt Cloud?

a. To customize the behavior of dbt in different environments.\
b. To create schemas in different colors.\
c. To configure databases only for production environments.\
d. To override the target name set in a developer’s profile.

Answer: a

##
What happens if a developer does not specify a custom schema name in dbt Cloud?

a. dbt creates an error and stops the build.\
b. dbt uses the default schema name configured for the environment.\
c. dbt generates a random schema name.\
d. dbt uses the schema name from the last successful run.

Answer: b

##
Which function in Jinja would you use to retrieve the environment variable `DBT_MY_ENV`?

a. `target.name`\
b. `schema_var`\
c. `env_var`\
d. `generate_env`

Answer: c
